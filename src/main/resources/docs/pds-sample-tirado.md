## Summary: Real-World Data Patterns

This sample demonstrates several critical edge cases:

| Pattern | Evidence | System Must Support |
|---------|----------|---------------------|
| **Multiple COLLEGE entries** | 2 colleges (PUP + Adamson) | ✓ Multiple rows per education level |
| **Multiple GRADUATE STUDIES** | 3 graduate programs (Law, MBA, Masters) | ✓ Multiple rows per level |
| **Ongoing education** | "PRESENT" with "THESIS COMPLETION ONGOING" | ✓ Handle "PRESENT" + blank year |
| **Fractional units** | "7.5 UNITS" | ✓ Support decimal units |
| **Present end date** | 18 work entries + several vol. work with "PRESENT" | ✓ "PRESENT" rendering |
| **Part-time employment** | 2 part-time entries with "601/unit", "501/unit" status | ✓ Part-time section separation |
| **Text-based hours** | "8-week online programme", "6-month online program" | ⚠️ Numeric `noHours` insufficient |
| **Long names** | 50+ character position titles, agency names | ✓ Wrap/font-size handling in Jasper |
| **Deceased family member** | Father marked "DECEASED" | ✓ Handle deceased status |
| **Optional fields** | Many "N/A" entries (PSN, agency no., salary, etc.) | ✓ Show "N/A" or blank appropriately |
| **Long address** | Multi-line residential address | ✓ Address wrapping |
