### III. EDUCATIONAL BACKGROUND

Education entries grouped by **Degree Level** (not chronologically):

| Level | Sub-fields | Notes |
|-------|-----------|-------|
| **Elementary** | Name of School, Basic Education/Degree/Course, Period From, Period To, Highest Level/Units Earned, Year Graduated, Scholarship/Academic Honors | One row only per form |
| **Secondary** | Same columns | One row only per form |
| **Vocational / Trade Course** | Same columns | One row only per form |
| **College** | Same columns | **Multiple rows possible** — e.g., two College entries (PUP + Adamson) |
| **Graduate Studies** | Same columns | **Multiple rows possible** — e.g., three Graduate Studies entries (Law, MBA, Masters in Int'l Health) |

**Column Details:**
- **Name of School** (write in full; do not abbreviate)
- **Basic Education/Degree/Course** (write in full; e.g., "BACHELOR OF SCIENCE IN BUSINESS MANAGEMENT (MARKETING MAJOR)")
- **Period From** (year, e.g., 1985)
- **Period To** (year, e.g., 1989; or "PRESENT")
- **Highest Level/Units Earned** (if graduated: blank; if not graduated: "21 UNITS" or "7.5 UNITS")
- **Year Graduated** (e.g., 1989; or blank if "PRESENT")
- **Scholarship/Academic Honors Received** (e.g., "ACADEMIC SCHOLARSHIP (FULL/PARTIAL)", "WITH HONORS", blank for "N/A")

**Special Cases:**
- If "upToPresent" = true, render "PRESENT" in the To column and leave Year Graduated blank
- Units earned can be fractional (e.g., "7.5 UNITS")

**Signature & Date Line**: Wet signature/e-signature/digital certificate accepted.

## Key 2025 Revision Changes vs. Prior Versions

1. **Item 10**: Renamed from "**SSS No.**" → "**UMID ID No.**"
2. **Item 13**: **New field** — "**PhilSys Number (PSN)**"
3. **Date Format**: All dates must be "**dd/mm/yyyy**" (e.g., 04/05/1968, not 05/04/1968)
4. **Signature Acceptance**: Explicitly states "wet signature/e-signature/digital certificate" accepted (except notary public section = wet only)
5. **GSIS Fields Removed**: GSIS BP No. and GSIS Policy No. no longer appear; only GSIS ID No. remains (in Item 10 area if at all)
6. **Sex at Birth**: Uses checkboxes for Male / Female; label explicitly says "Sex at Birth" (not "Gender")
7. **Background Image**: Updated blue form background with 2025 form branding

## Validation Checklist

When rendering a PDS report against this spec:

- [ ] All personal information (Items 1–21) present and formatted correctly
- [ ] All family background (Items 22–25) present; "DECEASED" shown for deceased parent
- [ ] All educational background grouped by level; multiple rows per level supported
- [ ] Date format is **dd/mm/yyyy** throughout
- [ ] UMID ID (item 10) and PhilSys PSN (item 13) both present on page 1
- [ ] Dates show "PRESENT" when applicable; units shown as "X UNITS" or fractional
- [ ] Work experience shows "PRESENT" for end date if current
- [ ] Questions 34–40 all present with Yes/No + if-yes details
- [ ] References limited to 3 rows; position title included
- [ ] Profile photo dimensions correct and visible
- [ ] Government ID combined date/place field correct (DATE / PLACE)
- [ ] All signatures/dates in correct format
- [ ] No GSIS BP/Policy No. fields present (2025 removal)
